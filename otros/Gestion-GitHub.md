# Gestion del Proyecto en GitHub

## Flujo recomendado

1. Crear una rama `feature/<nombre-corto>` por historia.
2. Vincular commits y Pull Requests con el Issue correspondiente.
3. Exigir CI verde antes de fusionar.
4. Solicitar al menos una revision de otro integrante.
5. Mover la tarjeta del GitHub Project al cambiar de estado.

## Issues iniciales sugeridos

- Validar recorrido ciudadano completo en Docker.
- Validar recorrido funcionario completo en Docker.
- Adjuntar evidencia responsive de movil, tablet y escritorio.
- Comparar implementacion final con Figma.
- Ejecutar y adjuntar reporte Cypress.
- Revisar secretos y credenciales antes de publicar.

## GitHub Project

Columnas sugeridas:

- Pendiente
- En curso
- En revision
- Terminado

Campos utiles:

- Prioridad
- Responsable
- Entrega
- Requisito EF1-EF6

## Discussion de cierre

Publicar una retrospectiva con:

- decisiones de arquitectura;
- riesgos encontrados;
- evidencia de seguridad y rendimiento;
- trabajo de cada integrante;
- mejoras futuras.
